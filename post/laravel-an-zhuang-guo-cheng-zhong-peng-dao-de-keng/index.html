Laravel安装过程中碰到的坑

2019-05-15

安装composer Windows端，下载并且运行Composer-Setup.exe，它将安装最新版本的 Composer ，并设置好系统的环境变量，因此你可以在任何目录下直接使用 composer 命令。

更换安装源 由于composer官方源被墙，导致接下来的安装速度将会极慢，所以可以考虑更换国内源，如：https://packagist.laravel-china.org/。 更换方式有两种：

  • 选项一、全局配置（推荐）
$ composer config -g repo.packagist composer https://packagist.laravel-china.org
  • 选项二、单独项目配置 如果仅限当前工程使用镜像，去掉 -g 即可，如下：
$ composer config repo.packagist composer https://packagist.laravel-china.org

创建一个新的 Laravel 项目 创建一个新的 Laravel 项目有两种方式，都由命令行执行的：第一种是通过全局的 Laravel 安装器，另一种是通过 Composer 的 create-project 命令。

  • 使用 Laravel 安装器安装 安装 Laravel 安装器很简单，在命令行执行以下命令即可（如果已经安装过，会自动进行更新）： $ composer global require laravel/installer 安装完成后，后续就可以通过 laravel new [项目名称] 来创建新的 Laravel 项目：
laravel new First_lv
  • 使用 Composer create-project 命令安装&创建
$ composer create-project laravel/laravel First_lv --prefer-dist

效果和上面使用安装器安装的一样，不过，使用这个方式安装的一个好处，可以安装旧版本的 Laravel 项目，比如要安装 5.6 版本的项目 First_lv_56 ，可以这么做：

$ composer create-project laravel/laravel First_lv_56 5.6.* --prefer-dist

安装至此，本以为运行命令行：php artisan serve 就能够见到Laravel安装成功的页面展示了，但是却依然出现了问题。

  • 500 server error 解决方法：将项目根目录下的.env.example重命名为.env，因为.env.example只是一个新创建项目的示例模板，而.env才是真正的配置文件。 再次启动服务器，又碰到了问题：
  • /storage/logs/.log/日志内容local.ERROR: No application encryption key has been specified. 解决方法：执行命令行：php artisan key:generate （生成项目key值 自动填到 .env文件中 APP_KEY） 再次启动服务器，终于见到了想见的Laravel安装成功页面。
